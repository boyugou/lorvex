# Design Philosophy

Canonical location: [`docs/vision/DESIGN_PHILOSOPHY.md`](../../../../docs/vision/DESIGN_PHILOSOPHY.md)

Lorvex presents a curated presentation over persisted planning state, not a raw table browser. The product should feel like a clean operating surface for the day, not just a passive persistence viewer.

The old Inbox UI/review surface has been removed from the active product model. AI-managed priority remains canonical for ranking, review, and focus decisions.

The schema-seeded `inbox` default list remains an internal bootstrap/default-routing artifact, not an active Inbox UI/review workflow.

Human override and power-user controls are shipped correction affordances for priority and list assignment when the AI default needs adjustment.
