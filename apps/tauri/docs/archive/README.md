Use current canonical docs instead:
- product direction: `docs/vision/`
- current design and behavior: `docs/design/`
- operational workflow and release gates: `docs/execution/`
- setup and install guidance: `docs/setup/`

Archived contents:
- `decisions/` — old reasoning logs and design notes
- `plans/` — one-off historical implementation plans retained only for reference
- `rfcs/` — superseded or milestone-era proposals

Removed archive sets:
- `optimal_arch/` — removed because it described a retired Tauri-owned
  CloudKit/iOS/App Store target architecture. Current architecture lives under
  `docs/design/`.
- `MASTER_BACKLOG.md` — removed because active planning no longer lives in
  repo-tracked backlog documents.

If a historical document still seems necessary for current understanding, its useful content should be folded back into a canonical doc rather than linked from current materials.
